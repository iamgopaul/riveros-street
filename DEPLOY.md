# Deploying to Vercel

Rivero's Street is a single Next.js app that serves three surfaces by hostname:

| Host                       | Serves        |
| -------------------------- | ------------- |
| `riverosstreet.com`        | Hub (home)    |
| `eat.riverosstreet.com`    | Food truck    |
| `shop.riverosstreet.com`   | Shop          |

Routing is handled by [`src/proxy.ts`](src/proxy.ts) (Next.js's proxy/middleware
convention) — no extra config needed. Vercel runs it automatically at the edge.

## 1. Import the project

1. Push this repo to GitHub/GitLab/Bitbucket.
2. In Vercel: **Add New → Project → Import** the repo.
3. Framework preset auto-detects **Next.js**. Build command `next build`,
   output handled automatically. Click **Deploy**.

The first deploy works immediately on the `*.vercel.app` URL (it shows the hub).

## Quick demo on one Vercel URL (no custom domain)

Vercel **won't** let you create `eat.<project>.vercel.app` / `shop.<project>.vercel.app`
— the `*.vercel.app` namespace is Vercel's. To demo all three surfaces on your
single project host, serve them by **path** instead and set base paths so the
in-surface links resolve. In **Settings → Environment Variables**, add:

```
NEXT_PUBLIC_HUB_URL=https://<project>.vercel.app
NEXT_PUBLIC_EAT_URL=https://<project>.vercel.app/eat
NEXT_PUBLIC_SHOP_URL=https://<project>.vercel.app/shop
NEXT_PUBLIC_EAT_BASE=/eat
NEXT_PUBLIC_SHOP_BASE=/shop
```

Redeploy, then browse:

- Hub:        `https://<project>.vercel.app`
- Food truck: `https://<project>.vercel.app/eat`
- Shop:       `https://<project>.vercel.app/shop`

When you later add a real domain (below), **remove the two `*_BASE` vars** and point
the `*_URL` vars at the subdomains — the app switches back to clean root-relative
links automatically.

## 2. Add the domains (real subdomains)

In **Project → Settings → Domains**, add all three:

- `riverosstreet.com` (apex)
- `www.riverosstreet.com` → redirect to apex (Vercel offers this automatically)
- `eat.riverosstreet.com`
- `shop.riverosstreet.com`

Then in your DNS provider:

- Apex `@` → Vercel (A record `76.76.21.21`, or follow Vercel's exact instructions).
- `www`, `eat`, `shop` → `CNAME` to `cname.vercel-dns.com`.

> A wildcard `*.riverosstreet.com` CNAME also works and covers future subdomains.

## 3. Environment variables

In **Project → Settings → Environment Variables** (Production + Preview):

### Cross-subdomain URLs (so links point at the right host)

```
NEXT_PUBLIC_HUB_URL=https://riverosstreet.com
NEXT_PUBLIC_EAT_URL=https://eat.riverosstreet.com
NEXT_PUBLIC_SHOP_URL=https://shop.riverosstreet.com
```

### Square payments (optional — card checkout)

Start in **sandbox**, then switch to production keys when ready. Without these,
checkout falls back to the WhatsApp "pay at the window" flow.

```
NEXT_PUBLIC_SQUARE_APP_ID=...        # from Square Developer dashboard
NEXT_PUBLIC_SQUARE_LOCATION_ID=...
NEXT_PUBLIC_SQUARE_ENV=production     # or sandbox
SQUARE_ACCESS_TOKEN=...              # SERVER secret — never NEXT_PUBLIC
SQUARE_LOCATION_ID=...
SQUARE_ENV=production                 # or sandbox
```

After changing env vars, **redeploy** (Vercel → Deployments → Redeploy) so they
take effect.

## 4. Verify

- `riverosstreet.com` → hub home, transparent nav, red logo.
- `eat.riverosstreet.com/menu` → orderable menu; add to cart → checkout.
- Share a link → the branded Open Graph card appears
  (generated by [`src/app/opengraph-image.tsx`](src/app/opengraph-image.tsx)).
- If Square is configured, run one sandbox order with test card
  `4111 1111 1111 1111`, any future expiry, CVV `111`, ZIP `94103`.

## Notes

- Local dev uses `*.localhost:6001`; no env vars are required locally.
- `next.config.ts` writes the build to `.next` on Vercel (and `.next-dev.nosync`
  locally to dodge iCloud sync corruption).
